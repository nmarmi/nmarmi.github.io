About Me

Hi, I am Nicolò Marmi, a student at Stony Brook University pursuing a double major in Honors Computer Science and Applied Mathematics.

My passion for technology began at a young age. When I was 9 I asked my dad if we could take apart our old computer before throwing it out (we did, it was a lot of fun!). Shortly after, I asked for Lego Mindstorms for Christmas, and built a robot that could follow the path I laid out with black tape. When I was 13 I started playing around with Scratch, and I built a couple of buggy, unfinished projects loaded with middle-school humor (I know, I was hilarious). Finally, at 17 I taught myself Python, and I fell in love with computer science and the problem-solving mindset that comes with it.

Since then, I have never stopped learning and exploring this field. I have enrolled in college for computer science, worked on a variety of projects, and gained both academic and professional experience. I am always excited to learn, build, and put myself to the test with a challenging problem.

Currently, I am working on my honors thesis on quantum ML with Brookhaven National Lab, and this summer I'm heading to Dublin to build NLP models to generate trading signals at Fineco Asset Management. I am really drawn to problems that intersect software and quantitative solutions, and I enjoy working in supportive, goal-oriented teams.

Some of my favorite projects

01 • GitHub

Quantum ML

Problem: quantum state tomography is expensive — full reconstruction requires exponentially many measurements as system size grows.

Stack: Python, PyTorch, NumPy, Qiskit, CUDA

Outcome: predicted unknown reduced density matrices from partial measurements, cutting tomography cost and accelerating quantum simulations.

02 • DraftIQ • DataAPI

DraftIQ + Player Data API

Problem: fantasy baseball drafts lack real-time player analytics and a reliable data API that teams can build on.

Stack: JavaScript, Node.js, Next.js, MongoDB Atlas

Outcome: full-stack app and licensed API handling 1000+ concurrent requests with average response time under 100ms, built by a team of 4 using Agile.

03 • GitHub

Playlister

Problem: build a full-stack playlist manager with a redesigned backend, improved database abstraction, and reliable unit test coverage.

Stack: JavaScript, Node.js, React, MongoDB

Outcome: clean client-server architecture with modular database layer and unit tests validating core functionality end-to-end.

04 • GitHub

Clip Extractor

Problem: manually cutting video sub-clips by face is slow, error-prone, and doesn't scale to large video libraries.

Stack: Python, face-recognition, OpenCV, MoviePy, Click

Outcome: automated face-based extraction with batch processing reduced manual editing to nearly 0%, with a lightweight CLI for reproducibility.

05

Algorithmic Trading Program

Problem: translating trading strategies from research into a system that can be backtested, extended, and validated reliably.

Stack: Python, Polars, TA-Lib

Outcome: modular backtesting framework validated through paper trading, achieving consistent positive returns with only 20% losing trades.

06

Listening Engine • Capco

Stack: Python, OpenAI API, Pydantic AI, Click
